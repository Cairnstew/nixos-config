# Windows DSC Sync ‚Äî Ongoing Nix‚ÜíWindows Config Management

Syncs the generated DSC v3 YAML config from NixOS to the Windows NTFS partition
whenever `/etc/dscnix/desktop.yaml` changes (i.e., on every `nixos-rebuild switch`).

## Options

| Option | Default | Description |
|--------|---------|-------------|
| `my.services.windowsDscSync.enable` | `false` | Enable DSC sync |
| `my.services.windowsDscSync.windowsPartition` | `"/dev/disk/by-partlabel/Windows"` | Windows partition device |
| `my.services.windowsDscSync.mountPoint` | `"/mnt/windows-dsc-sync"` | Temp mount point |
| `my.services.windowsDscSync.windowsTargetDir` | `"NixOS"` | Target dir (relative to Windows root) |

## Behavior

A systemd path unit watches `/etc/dscnix/desktop.yaml`. When the file changes:

1. Mounts the Windows partition (read-write, falls back to read-only)
2. Writes the updated DSC YAML to `C:\NixOS\dsc-configuration.yaml`
3. Writes `C:\NixOS\apply-dsc.ps1` bootstrap script (if not present)
4. Clears the applied-state JSON if config hash changed, forcing re-apply
5. Unmounts and exits

On the next Windows boot, the `apply-dsc.ps1` bootstrap script (triggered by a
scheduled task set up during initial install) detects the changed YAML and
runs `dsc config set` to apply it.

## Requirements

- Initial Windows install must register a scheduled task that runs
  `C:\NixOS\apply-dsc.ps1` on system startup (handled by the `autounattend.xml`
  generated by the `windows-installer` module)
- Windows partition must be accessible (not hibernated ‚Äî disable fast startup
  in Windows power settings for reliable read-write access)
